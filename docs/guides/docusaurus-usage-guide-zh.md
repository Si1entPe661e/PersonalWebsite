## 3. 如何新增一篇文档

### 3.1 先决定放在哪个分区

例如：

- 经济学内容放进 `economics-docs`
- 数学内容放进 `mathematics-docs`
- 编程和工作流内容放进 `programming-docs`

### 3.2 在对应目录里新建文件

比如你要新增一篇编程笔记：

`/Users/lizhicheng/Desktop/PersonalWebsite/notes-docs/programming-docs/my-new-note.md`

### 3.3 最简单的文档格式

你可以直接写：

```md
---
title: My New Note
---

# My New Note

This is a new note.

## Section One

Write your content here.
```

保存后，Docusaurus 就会把它当成一篇新文档。

## 4. `.md` 和 `.mdx` 有什么区别

### `.md`

适合：

- 纯文字
- 普通标题
- 列表
- 表格
- 代码块

也就是说，大多数笔记都可以直接用 `.md`。

### `.mdx`

适合：

- 想在文档里插入 React 组件
- 想写更复杂的交互内容
- 想插入更高级的自定义内容块

如果你只是写普通笔记，优先用 `.md`。  
只有在你明确知道自己需要更复杂效果时，再用 `.mdx`。

一句话记忆：

- 普通笔记：用 `.md`
- 特殊展示：用 `.mdx`

---

## 5. 一个标准的 `.md` 文档怎么写

推荐你每篇文档都按这个结构写：

```md
---
title: Difference-in-Differences Checklist
description: A short working checklist for applied DID work.
---

# Difference-in-Differences Checklist

This note collects the basic checks I want to review before trusting a DID design.

## 1. Identification target

Explain what you are trying to identify.

## 2. Treatment timing

Explain how timing is defined.

## 3. Parallel trends

Describe the assumption and how you will check it.
```

这里最上面的：

```md
---
title: ...
description: ...
---
```

叫做 `front matter`。

你可以把它理解成：

- `title`：这篇文档的标题
- `description`：这篇文档的简短介绍

## 6. 一个最简单的 `.mdx` 文档怎么写

如果你真的要用 `.mdx`，可以从这个模板开始：

```mdx
---
title: My MDX Note
description: A note with custom content.
---

# My MDX Note

This is a normal paragraph.

## A code example

```python
print("hello world")
```
```

如果你现在还不确定 `.mdx` 的意义，那就先不要用，继续用 `.md` 就够了。

## 7. 如何控制左侧侧边栏

当前项目里，三个分区都用的是“自动生成侧边栏”。

例如：

`/Users/lizhicheng/Desktop/PersonalWebsite/notes-docs/sidebarsProgramming.ts`

现在内容是：

```ts
import type {SidebarsConfig} from '@docusaurus/plugin-content-docs';

const sidebars: SidebarsConfig = {
  programmingSidebar: [{type: 'autogenerated', dirName: '.'}],
};

export default sidebars;
```

这表示：

- Docusaurus 会根据目录结构自动生成左侧栏
- 你新增文件后，通常不需要手动一个个写进去

也就是说：

- 如果你只是正常新增文档，通常不用改侧边栏文件

## 8. 如何让侧边栏出现折叠分类

最简单的方式，不是去写很复杂的代码，而是：

### 方法 1：用文件夹分组

比如你在 `programming-docs` 下面建一个文件夹：

`workflow`

里面放：

- `intro.md`
- `claude-code-setup.md`
- `reproducible-research-environment.md`

目录会变成：

```text
programming-docs/
  workflow/
    intro.md
    claude-code-setup.md
    reproducible-research-environment.md
```

这样自动生成的侧边栏通常就会把 `workflow` 当成一个分类。

如果当前配置允许折叠，这个分类就会显示成可展开/可收起。

## 9. 当前项目里“侧边栏可折叠”是怎么开的

在这个文件：

`/Users/lizhicheng/Desktop/PersonalWebsite/notes-docs/docusaurus.config.ts`

你会看到这一段：

```ts
docs: {
  sidebar: {
    hideable: true,
    autoCollapseCategories: false,
  },
},
```

它的意思是：

- `hideable: true`
  左侧整栏可以被用户隐藏

- `autoCollapseCategories: false`
  分类不会自动全部折起来

如果你以后想继续保留“侧边栏可收起”的体验，这一段不要删。

## 10. 如果你想手动写侧边栏，而不是自动生成

你也可以不用 `autogenerated`，改成手动写。

比如把：

```ts
const sidebars: SidebarsConfig = {
  programmingSidebar: [{type: 'autogenerated', dirName: '.'}],
};
```

改成：

```ts
const sidebars: SidebarsConfig = {
  programmingSidebar: [
    'intro',
    {
      type: 'category',
      label: 'Workflow',
      collapsible: true,
      collapsed: false,
      items: [
        'claude-code-setup',
        'reproducible-research-environment',
      ],
    },
  ],
};
```

这里的意思是：

- `type: 'category'`
  这是一个分组

- `label`
  左侧栏显示的名字

- `collapsible: true`
  这个分组可以折叠

- `collapsed: false`
  默认先展开

## 11. 什么时候该用“自动侧边栏”，什么时候该用“手动侧边栏”

### 用自动侧边栏

适合：

- 文档数量不多
- 你想省事
- 目录结构本身就很清楚

### 用手动侧边栏

适合：

- 你想自己控制显示顺序
- 你想自定义分类名字
- 你想精确控制哪些内容可折叠

一句话建议：

- 刚开始：优先自动生成
- 文档很多以后：再改成手动

## 12. 文档标题和文件名怎么对应

文件名会影响 URL。

例如文件：

`claude-code-setup.md`

通常对应地址会像：

`/programming/claude-code-setup`

所以建议文件名尽量：

- 用英文
- 用短横线连接
- 不要太长

标题则写在文档里的 `title` 或正文 `# 标题` 中。

也就是说：

- 文件名负责网址
- 标题负责显示给读者看

## 14. 最常见的 3 种修改

### 情况 1：新增一篇笔记

做法：

1. 进入对应 docs 目录
2. 新建一个 `.md` 文件
3. 写标题和正文
4. 启动本地服务查看效果

### 情况 2：想把文档分组

做法：

1. 先尝试用文件夹整理
2. 如果不够，再改 `sidebars*.ts`

### 情况 3：想让左侧分组可以折叠

做法：

1. 手动写 sidebar category
2. 设置：
   - `collapsible: true`
   - `collapsed: false` 或 `true`
